Pain Points and Solutions

Shrimp Task Manager is designed to solve the three core pain points faced by AI programming assistants in task management.

Memory Loss

AI assistants lack cross-conversation task memory capabilities, making it impossible to track long-term task progress, leading to repeated explanations of the same requirements, wasting time and resources.

Task Memory Function

Automatically save execution history, provide long-term memory capabilities, and enable AI assistants to remember previous task progress and seamlessly continue unfinished tasks.

Structural Chaos

Complex tasks lack systematic management, leading to inefficiency, missing dependency management, chaotic execution of subtasks, and difficulty tracking overall progress.

Structured Task Decomposition

Automatically decompose complex tasks into manageable subtasks, establish clear dependencies, provide an orderly execution path, and ensure efficient completion.

Repetitive Work

Unable to effectively utilize past experiences and solutions, starting from scratch in every conversation, lacking a system for knowledge accumulation and experience reference.

Knowledge Accumulation and Experience Reference

Automatically record successful solutions, build a task knowledge base, support quick reference for similar tasks, and achieve experience accumulation and knowledge reuse.

Core Features

Shrimp Task Manager Provides six core features to help you efficiently manage, execute, and track complex tasks.

Intelligent Task Planning and Analysis

Generate structured task plans through in-depth analysis of requirements and constraints. Automatically assess scope, risks, and priorities, providing rational and comprehensive implementation strategies.

Automatic Task Decomposition and Dependency Management

Intelligently decompose complex tasks into manageable subtasks, identify dependencies between tasks, establish optimized execution paths, and avoid resource conflicts and execution bottlenecks.

Execution Status Tracking

Monitor the execution status of each task in real-time, provide visual progress displays, automatically update dependency statuses, and deliver detailed execution reports upon task completion.

Task Integrity Verification

Thoroughly check task completion, ensure all requirements and standards are met, provide verification reports and quality assessments, and ensure outputs meet expected requirements.

Task Complexity Assessment

Evaluate task complexity based on multidimensional criteria, provide resource requirement estimates, identify high-risk components, and help allocate resources and time effectively.

Task Memory Function

Provide cross-session task memory capabilities, automatically save execution history and context, allow tasks to be resumed and continued at any time without the need to repeatedly explain requirements.

Task Planning and Decomposition Process

This example shows how to use MCP Shrimp Task Manager to plan and decompose complex tasks. The entire process includes four main steps:

  1. Task Planning - Initialize and plan tasks in detail, clarify goals and success criteria
  2. In-depth Analysis - Gain in-depth understanding of tasks, analyze technical feasibility and potential challenges
  3. Solution Reflection - Critically review analysis results, optimize proposals
  4. Task Decomposition - Break down complex tasks into manageable subtasks

With this approach, you can transform complex large tasks into structured, executable work units while maintaining an overall perspective.

Task Execution and Completion Process

This example shows how to execute and complete planned tasks. The entire process includes four main steps:

  1. Task List - Query the pending task list to understand current status
  2. Task Execution - Execute selected tasks according to the predetermined plan
  3. Result Verification - Verify task completion and ensure quality standards are met
  4. Task Completion - Formally mark tasks as completed and generate reports

With this approach, you can systematically execute tasks and ensure each step meets the expected quality standards, ultimately completing the entire workflow.

Tip

The workflow above is not fixed. The Agent will iterate through different steps repeatedly based on analysis until the expected effect is achieved.

Prompt Customization Feature

Customize system prompts through environment variables, customize AI assistant behavior without modifying code

Feature Overview

Prompt customization allows users to adjust AI assistant behavior through environment variables, providing two customization methods: completely overriding the original prompt or appending content to the existing one.

Main Benefits

  • Personalized customization: Adjust system behavior according to specific project or domain requirements
  • Efficiency improvement: Optimize for repetitive task types, reduce redundant instructions
  • Brand consistency: Ensure output content conforms to organization's style guidelines and standards
  • Professional adaptability: Adjust professional terminology and standards for specific technical fields or industries
  • Team collaboration: Unify prompts used by team members, ensure consistent working methods

Usage Methods

Environment Variable Configuration

Set environment variables to customize prompts for various functions, using specific naming conventions: 

# Override Mode: MCP_PROMPT_[FUNCTION_NAME]
# Append Mode: MCP_PROMPT_[FUNCTION_NAME]_APPEND

# Example:
MCP_PROMPT_PLAN_TASK="Custom task planning prompt"
MCP_PROMPT_EXECUTE_TASK_APPEND="Additional task execution guidance"

Cursor IDE Configuration

If you use Cursor IDE, you can integrate Shrimp Task Manager into your development environment.

Your project directory/.cursor/mcp.json 
{
  "mcpServers": {
    "shrimp-task-manager": {
      "command": "node",
      "args": [
        "/path/to/mcp-shrimp-task-manager/dist/index.js"
      ],
      "env": {
        "DATA_DIR": "/path/to/project/data"
      }
    }
  }
}

Important Note

Must use absolute paths: Please ensure DATA_DIR configuration uses absolute paths rather than relative paths, otherwise data may not load correctly

Prompt Configuration Instructions

Shrimp Task Manager supports two modes:

  • TaskPlanner: Suitable for initial task planning and complex task decomposition, AI assistant plays the role of task planner. 
    
You are a professional task planning expert. You must interact with users, analyze their needs, and collect project-related information, ultimately using "plan_task" to create tasks. When tasks are created, you must summarize and inform users to use "TaskExecutor" mode for task execution.
You must focus on task planning and are prohibited from using "execute_task" to execute tasks.
SERIOUS WARNING: You are a task planning expert. You cannot directly modify code. You can only plan tasks, and you cannot directly modify code. You can only plan tasks.
  • TaskExecutor: Suitable for executing predefined tasks, AI assistant plays the role of execution expert. 
    
You are a professional task execution expert. When users specify tasks to execute, use "execute_task" to execute tasks.
When no task is specified, use "list_tasks" to find unexecuted tasks and execute them.
When execution is complete, you must summarize and inform users of the conclusion.
You can only execute one task at a time. When a task is completed, do not proceed to the next task unless the user explicitly instructs you to do so.
If the user requests "continuous mode", execute all tasks in sequence.

You can use Custom modes in Cursor settings to customize modes to adapt to different work scenarios.
